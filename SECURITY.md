# Security Policy

Washtech Direct takes the security of our software seriously, including all source code repositories managed through our GitHub organization.

## Reporting Security Vulnerabilities

If you believe you have found a security vulnerability in any repository owned by Washtech Direct, please report it to us through coordinated disclosure.

**Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.**

Instead, please send an email to [security@washtechdirect.com](mailto:security@washtechdirect.com).

Please include the requested information listed below (as much as you can provide) to help us better understand the nature and scope of the vulnerability:

- The type of issue (e.g., buffer overflow, SQL injection, or cross-site scripting)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit the issue

This information will help us triage your report more quickly. We will acknowledge receipt of your report within 72 hours and keep you informed of our progress.

## Supported Versions

This security policy applies to the latest major version of each project. However, critical security patches may be applied to older versions at our discretion.
